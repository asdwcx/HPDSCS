## 创建群聊流程可视化

### 🎯 **标准创建群聊流程**

```
时间轴    用户A(Alice)                服务器                     用户B(Bob)    用户C(Charlie)
 T1       creategroup:技术交流群       接收请求                       
          :1002,1003               |                              
          |                         |                              
          |-----------------------> | 验证群名不重复                   
          |                         | 验证成员ID有效性                
          |                         | 创建群组记录                   
          |                         | 设置创建者为群主                
          |                         |                              
 T2       等待创建结果...              | 添加群成员关系                  
          |                         | 分配成员角色                   
          |                         |                              
 T3       [成功] 群聊创建成功！        | --------群邀请通知-------> [群邀请] Alice邀请你   [群邀请] Alice邀请你
          群ID: 10001              |                         | 加入"技术交流群"      | 加入"技术交流群"  
          群名: 技术交流群            |                         |                    |
          成员: Alice, Bob, Charlie  |                         |                    |
          |                         |                              
 T4       groupchat:10001:大家好！    | --------群消息广播------>   [群消息] Alice: 大家好！ [群消息] Alice: 大家好！
```

### 🔄 **数据流转详解**

#### **消息类型序列：**
```
1. CREATE_GROUP_MSG     (A→服务器)   创建群聊请求
2. CREATE_GROUP_MSG     (服务器→A)   创建结果通知
3. GROUP_NOTIFY         (服务器→B,C) 群邀请通知
4. ONE_CHAT_MSG         (任意→服务器) 群消息
5. ONE_CHAT_MSG         (服务器→群内) 群消息广播
```

#### **数据库变化：**
```sql
-- 1. 在allgroup表中创建群组记录
INSERT INTO allgroup VALUES (10001, '技术交流群', '技术交流和讨论', 1001);

-- 2. 在groupuser表中添加群成员关系
INSERT INTO groupuser VALUES (10001, 1001, 'creator');  -- Alice为创建者
INSERT INTO groupuser VALUES (10001, 1002, 'normal');   -- Bob为普通成员
INSERT INTO groupuser VALUES (10001, 1003, 'normal');   -- Charlie为普通成员
```

### 📊 **创建群聊状态机**

```
开始创建
   |
   | creategroup:群名:成员列表
   ↓
验证阶段
   |
   +----群名重复------→ 创建失败 → 提示重新命名
   |
   +----成员ID无效----→ 创建失败 → 提示检查成员ID
   |
   +----验证通过------→ 执行创建
   |
   ↓
创建群组记录
   |
   ↓
添加成员关系
   |
   ↓
发送通知
   |
   ↓
创建完成 → 可以群聊
```

### 🛠 **创建群聊命令详解**

#### **命令格式：**
```bash
creategroup:群名称:成员ID列表

示例：
creategroup:技术交流群:1002,1003,1004
creategroup:项目讨论组:1005
creategroup:朋友圈:1002,1003,1004,1005,1006
```

#### **参数说明：**
- **群名称**: 1-20个字符，不能与现有群重名
- **成员ID列表**: 用逗号分隔的用户ID，最多50人
- **创建者**: 自动成为群主，无需在成员列表中指定

### 🎯 **特殊情况处理**

#### **1. 群名重复**
```bash
>> creategroup:技术交流群:1002,1003
[错误] 群聊创建失败：群名称"技术交流群"已存在，请换一个名称
```

#### **2. 成员ID无效**
```bash
>> creategroup:新项目组:1002,9999,1004
[错误] 群聊创建失败：用户ID 9999 不存在
有效成员：1002, 1004
请检查后重新创建
```

#### **3. 成员过多**
```bash
>> creategroup:大群:1002,1003,...(超过50人)
[错误] 群聊创建失败：群成员数量超过限制(50人)
当前指定成员：52人
```

#### **4. 空成员列表**
```bash
>> creategroup:个人笔记:
[成功] 群聊创建成功！(仅群主)
群ID: 10002
群名: 个人笔记
成员: Alice(1001) [群主]
```

### 📋 **群管理功能扩展**

#### **查看群信息：**
```bash
>> groupinfo:10001
群ID: 10001
群名: 技术交流群
群主: Alice(1001)
创建时间: 2025-01-15 14:30:22
成员数: 3人
成员列表:
- Alice(1001) [群主] [在线]
- Bob(1002) [成员] [在线]  
- Charlie(1003) [成员] [离线]
```

#### **邀请新成员：**
```bash
>> addusertogroup:10001:1005
[成功] 用户 David(1005) 已被添加到群聊 "技术交流群"
[群通知] Alice 邀请 David 加入了群聊
```

#### **退出群聊：**
```bash
>> quitgroup:10001
[确认] 你已退出群聊 "技术交流群"
[群通知] Bob 退出了群聊
```
